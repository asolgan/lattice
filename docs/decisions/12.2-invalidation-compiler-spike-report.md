## GO / NO-GO RECOMMENDATION FOR 12.3

**GO — with ONE mandatory design correction the spike discovered and validated.**

The brief's claim ("the machinery already exists on both sides") holds, **but the
naive composition of it does not**. Flattening the full AST into a single
`simple.QueryPlan.Steps` slice and reusing the simple engine's verbatim reverse
walk is **UNSOUND for multi-branch lenses** such as `capabilityEphemeral`. The
spike caught this empirically — the no-missed-anchor reproject-and-diff failed on
the `capabilityEphemeral` / `link_assignedTo` event, dropping the manager anchor
(a **missed revocation** on the auth plane).

The fix lives in the **compiler** (the new code 12.3 writes), not in the reverse
walk: segment the AST into a **forest of per-branch linear anchor→leaf chains**,
run the (unchanged) reverse walk over **each** branch, and union the results.
With that correction, the oracle holds for all three event kinds on both lenses:
subset (strict on every event), zero missed anchors.

So: **GO for 12.3's D-PIPELINE option-1 approach (compiled invalidation plan +
`projectionKind: actorAggregate` marker), with the per-branch segmentation as a
hard requirement, not an optimization.** A single-linear-plan compiler is a
latent missed-revocation bug and must not ship.

### What 12.3 inherits / must do for real (not the spike copy)

1. **Wire the REAL reverse walk, not this spike's copy.** `reverse_copy.go` is a
   VERBATIM copy of `internal/refractor/ruleengine/simple/{evaluator,plan}.go`
   (`reverseTraverse`/`walkBackToAnchor`/`filterEdges`/`reverseDirection`),
   headed as throwaway. 12.3's production wiring must **export/relocate the real
   functions** (or move the compiler into `package simple`) and call those — it
   does **NOT** ship this copy. The copy exists only so the spike exercises the
   exact production algorithm without editing production.
2. **Per-branch segmentation is mandatory.** Build the traversal forest by
   following From→To variable chaining from the anchor (see `compiler.go`
   `buildBranches`); run the reverse walk per branch and union. Do not flatten to
   one `Steps` slice.
3. **Backfill re-referenced-anchor labels.** A re-referenced anchor node in an
   `OPTIONAL MATCH` (e.g. `(identity)<-[:reportsTo]-(report:identity)…` in
   `capabilityEphemeral`) carries no inline label, so `NodePattern.Label` is
   empty. The compiler must recover it from the variable's introducing binding
   (the spike's `labelOf` map), or the reverse-walk direction/label matching is
   wrong.
4. **Enforce fail-closed on the auth plane.** When a lens marked
   `projectionKind: actorAggregate` (e.g. `capabilityEphemeral`) contains a
   construct the compiler cannot prove subset-safe, 12.3 must **fail activation
   closed** (refuse to register the lens), not silently fall back to broad BFS.
   Non-security lenses may fall back to BFS. The spike only records this policy.
5. **Compile the LIVE specs, not a snapshot.** The spike pins the two cypher
   strings verbatim (snapshot 2026-06-14, `specs.go`). 12.3 compiles the live
   `packages/orchestration-base/lenses.go` specs through `full.Engine.Parse`.
6. **Validate the punted constructs** (collect/`+`, WITH, var-length) against the
   covered/not-covered table below before relying on them in production.

## How the spike works (the oracle)

For each lens (`capabilityEphemeral`, `myTasks`) and each CDC event kind
(vertex, link, aspect):

1. **Compile** the real cypher: `full.Engine.Parse(spec)` → full AST →
   `CompilePlan` walks it into a **forest** of per-branch linear
   `simple.QueryPlan`s (the new capability — the simple engine's own `Compile`
   never consumes the full AST).
2. **Compiled set** = union over branches of the **verbatim** simple-engine
   reverse walk (`reverseTraverse`) from the changed non-anchor node.
3. **BFS oracle** = `pipeline.ActorEnumerator.Enumerate` seeded the way the
   production pipeline seeds it (vertex event → the vertex; link event → BOTH
   endpoints, per `evaluate.go evaluateLinkFanOut`; aspect event → parent vertex).
4. **Assert (a)** compiled ⊆ BFS, strict on ≥1 event.
5. **Assert (b)** no missed anchor — the REAL reproject-and-diff (decision #3,
   higher bar): reproject **every BFS actor** through `full.Engine.ExecuteWith`
   before/after the fixture mutation, diff the projected rows, and assert every
   actor whose output changed is in the compiled set.
6. **Record (c)** `len(BFS) − len(compiled)`.

**No-missed-anchor bar hit:** the REAL reproject-and-diff (decision #3's literal
intent), not the conservative predicate fallback. The full-engine evaluate over
the BFS superset wired cleanly using the existing
`full.Engine.ExecuteWith(ctx, cr, EventContext{Parameters}, adjKV, coreKV)` entry
point and the production contract-test fixture shape.

## The fixture graph

```
mgr (identity, manager)
rep (identity) -[reportsTo]-> mgr            # mgr inherits rep's tasks (delegation)
bystander (identity)                          # the over-reprojection case

taskDirect (task) -[assignedTo]-> mgr         # status open / expiresAt future
  taskDirect -[forOperation]-> op
  taskDirect -[scopedTo]-> tgt
taskRep (task) -[assignedTo]-> rep            # delegated to mgr via reportsTo

bystander -[watches]-> op                      # shares op via a NON-lens relation
```

`bystander` is reachable from any task by **undirected** BFS through the shared
`op` vertex, but never by the **directed, named** lens paths — so it is the
real over-reprojection the compiled plan eliminates (`compiled ⊊ BFS`).

## Compiled step lists (actual `[]TraversalStep`, per branch)

**myTasks** (2 branches):

- branch 0: `(identity)<-[assignedTo]<-(task)`, `(task)->[forOperation]->(op)`
- branch 1: `(identity)<-[assignedTo]<-(task)`, `(task)->[scopedTo]->(tgt)`

**capabilityEphemeral** (4 branches — direct + delegation, each split by the
two leaf hops):

- branch 0: `(identity)<-[assignedTo]<-(task)`, `(task)->[forOperation]->(op)`
- branch 1: `(identity)<-[assignedTo]<-(task)`, `(task)->[scopedTo]->(tgt)`
- branch 2: `(identity)<-[reportsTo]<-(report:identity)`, `(report)<-[assignedTo]<-(task2)`, `(task2)->[forOperation]->(op2)`
- branch 3: `(identity)<-[reportsTo]<-(report:identity)`, `(report)<-[assignedTo]<-(task2)`, `(task2)->[scopedTo]->(tgt2)`

Direction mapping is load-bearing (the reverse walk reverses it):
`full.DirIn → simple.Inbound`, `full.DirOut → simple.Outbound`. Both lenses'
`<-[:assignedTo]-` / `<-[:reportsTo]-` hops are `DirIn`; the leaf
`-[:forOperation]->` / `-[:scopedTo]->` hops are `DirOut`.

|---|---|---|---|---|---|---|
| myTasks | vertex_task | 3 | 1 | 2 | yes | 1 |
| myTasks | link_assignedTo | 3 | 1 | 2 | yes | 1 |

| capabilityEphemeral | vertex_task | 3 | 1 | 2 | yes | 1 |
| capabilityEphemeral | link_assignedTo | 3 | 2 | 1 | yes | 2 |
| capabilityEphemeral | aspect_task_data | 3 | 1 | 2 | yes | 1 |

The `capabilityEphemeral` / `link_assignedTo` row is the load-bearing one: a
change to the delegated task's `assignedTo` link changes **two** actors' output
(`rep` directly, `mgr` via delegation) and the corrected forest compiler catches
**both** (`compiled=2`, `changed-output=2`). The pre-fix single-linear-plan
compiler caught only `rep` — the missed-anchor bug.

## No-missed-anchor result

For every event on both lenses, `{actors whose reprojected output changed} ⊆
{compiled affected-anchor set}`. Zero missed anchors with the per-branch
compiler. (The pre-fix flat-plan compiler FAILED this check on
`capabilityEphemeral`/`link_assignedTo` — the empirical catch that drove the
design correction.)

## Covered vs NOT-covered openCypher constructs

| construct | covered? | reasoning / subset-safety |
|---|---|---|
| `MATCH` / `OPTIONAL MATCH` | ✅ | Each pattern's hops become traversal steps; OPTIONAL vs required affects only the anchor pick, not reachability for invalidation. |
| node labels | ✅ | Carried into `From/ToLabel`; the reverse walk matches the changed node by label. Re-referenced anchors need label backfill (see "what 12.3 must do"). |
| relationship names + directions | ✅ | Carried into `EdgeType` + `Direction`; direction mapping is exact and load-bearing. |
| variable-length hops within the cap (`DefaultActorMaxDepth=10`) | ✅ (bounded) | The branch builder caps branch length at 10 (mirrors the BFS/executor cap). Neither real lens uses `*` quantifiers; an unbounded `*` must be clamped to the cap and is then still a bounded subset of the BFS. |
| **multi-branch** patterns (disjoint paths off the anchor) | ✅ **only via per-branch segmentation** | A single flat `Steps` slice is UNSOUND (interleaves branches → missed anchor). MUST compile to a forest and walk each branch. **This is the spike's central finding.** |
| narrowing `WHERE` (`status='open'`, `expiresAt > $now`) | ✅ (ignored, safe) | A WHERE that only NARROWS the matched set is safe to ignore for invalidation: dropping it can only ENLARGE the compiled affected-anchor set toward (never past) the BFS superset, so it never MISSES an anchor — at worst it reprojects an actor whose output didn't change (a harmless over-reprojection the reproject-and-diff tolerates). Both lenses' WHEREs are pure narrowing filters. |
| simple path-preserving `WITH` | ✅ (treat as pass-through) | A WITH that only forwards bindings (no aggregation that changes the source set, no computed reachability) preserves the traversal paths, so steps compile unchanged. Neither lens uses WITH; supported in principle. |
| `collect(...)` / `collect(...) + collect(...)` in RETURN | ✅ (irrelevant) | Aggregation shapes OUTPUT, not reachability. The compiler does not consult RETURN; invalidation is purely about which anchors to reproject. |
| **broadening `WHERE`** (e.g. `OR` pulling in an unrelated path; a predicate the compiler can't bound) | ❌ NOT covered | Could ADD reachable anchors the directed steps don't capture → could MISS an anchor. NOT subset-safe → fail-closed for auth lenses. |
| **undirected hop** (`-[:t]-`, `full.DirBoth`) | ❌ NOT covered for auth | The reverse walk maps it to `Both`, which is sound but defeats the precision win and, combined with synthetic edges, can broaden. Neither real lens uses it; flag for fail-closed on auth lenses. |
| aggregation that changes the SOURCE set (`collect` feeding a later MATCH, `count` gating a path) | ❌ NOT covered | Changes which paths exist → reachability is data-dependent → not statically subset-provable. |
| computed / synthetic relationships, `UNION`, subqueries, pattern predicates in WHERE that assert existence of OTHER paths | ❌ NOT covered | Reachability not derivable from the static step list → not subset-provable. |

## Fallback policy (recorded; 12.3 enforces)

- **Non-security projections:** on encountering an uncovered construct, fall back
  to broad `ActorEnumerator` BFS reprojection. Correct (BFS is a sound superset),
  just less precise.
- **Auth-plane `actorAggregate` lenses** (e.g. `capabilityEphemeral`): MUST
  compile fully, or **fail activation closed** (refuse to register the lens). A
  silent BFS fallback is acceptable for correctness but a compile FAILURE on a
  security lens must be loud — a missed-revocation risk is never traded for
  availability. 12.3 owns this enforcement; the spike only records the policy.
