## Findings

1. **[MAJOR] A permanently-unresolvable pattern/op reference hot-loops on plain `Nak` and never surfaces to Health.**
   `strategist.go:84-89` and `:119-122` classify an unresolved `meta.loomPattern` / op-meta reference as `errTransient`; `evaluator.go:116-119` returns plain `substrate.Nak` (immediate redelivery). Correct for the boot-replay race, but a playbook typo (`"pattern": "onbaording"`) never resolves: the consumer spins in an immediate-redelivery loop indefinitely (only Warn logs), and — because the transient path bypasses `alert()` — this config-error class never reaches Health KV. That is both the hot-loop posture 8.5's Ruling 4 explicitly engineered out of loom's relay (substrate provides `NakWithDelay` + the 5s default floor precisely for this; the story's "no exponential backoff" pin does not forbid the fixed floor loom already uses) and a gap in the FR29 never-silently-drop discipline the story invokes for every other config error. Recommend: return `NakWithDelay`, and surface a keyed Health issue while the reference stays unresolved (clearing on resolution, as the other issue keys already do).

2. **[MINOR] The `NumDelivered` retry-disambiguation branch (Deviation 1) ships with zero test coverage.**
   `evaluator.go:139-145` is the one novel decision point the dev invented to reconcile AC5's "retry re-publishes the SAME requestId" with §10.8's "create loses → drop", and no e2e or unit test drives the redelivery path (publish-failure → Nak → mark-exists re-fire → identical requestId). The story only mandated unit-testing the derivation (done), so this is not an AC violation — but an untested adjudication-bearing branch on the dispatch-idempotency seam deserves a follow-up test (a fake-publish-failure harness or a handler-level unit test with a constructed `Message{NumDelivered: 2}`).

3. **[MINOR] Contract #5 shape nits, all inherited verbatim from loom's accepted `health.go`.**
   (a) Status stays `"healthy"` on every tick even while error-severity issues are present (health.go:194 + 213-222) — §5.3 ties non-empty `issues` to `degraded`/`unhealthy`, and weaver (unlike loom) now routes error-class config alerts through `issues`, making the mismatch material to operators; (b) the final status `"shutdown"` is not a §5.3 enum value (`shuttingDown`); (c) heartbeat writes carry no TTL (§5.6 expects TTL ≈ 100s so dead instances purge). AC8 demanded "mirroring `internal/loom/health.go`", which this does exactly — so these are platform-wide debts, not 9.1 defects. Suggest a small cross-component follow-up rather than a 9.1 fix.

4. **[MINOR] Story/epic/AC-number references in production comments.**
   `state.go:58` ("…is Story 9.2's"), `actuator.go:17` ("AC #9"), `evaluator.go:49` ("AC 7"), `health_sink.go:25` ("AC 8"), `strategist.go:67` ("AC 4"), `strategist.go:69/170-171` ("Epic 10"). None narrate a prior state (the CLAUDE.md letter), and loom set the identical precedent (`loom/actuator.go:17` "AC #8") — but "Story 9.2's" is literally the `// Story N.N` pattern CLAUDE.md names as the most-violated rule, and AC numbers go stale the moment the story closes. Recommend rephrasing to contract cites ("the §10.3 lease/sweep is not implemented here") at Winston's discretion; consistency with loom argues either both stay or a sweep fixes both.

5. **[INFO] Mark residue on target removal.** Tombstoning a target tears down its consumer, so its existing `weaver-state` marks are never level-reconciled away (no watch updates arrive); a re-install self-heals via `DeliverLastPerSubject` replay + the clearing pass, and a permanent removal leaves orphan marks for 9.2's reconciler. Within the accepted D2 interim posture, but this specific case is not named in the Completion Notes' stray-mark disclosure (which covers only the dropped-column case) — worth a line in Winston's 9.2 handoff.

6. **[INFO] The Reset branch (engine.go:276-289) is mechanically unreachable today and untested** — identical to the disclosed gap 8.5's review graded PARTIAL and Winston accepted; the spec diff is written generically so a future spec-field change is caught. No new exposure.

7. **[INFO] Malformed row keys (non-NanoID entity segment) are dropped with a Warn log only** (evaluator.go:21-23) — contract-faithful (§10.2 mandates the NanoID segment) and not redelivery-fixable, but visibility is log-only, not a Health issue. Acceptable for an internal bucket only Weaver and Lens projections write; folded here for completeness.
